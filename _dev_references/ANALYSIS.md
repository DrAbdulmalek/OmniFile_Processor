## 1. 9router - بوابة توجيه الذكاء الاصطناعي

**المستودع:** https://github.com/decolua/9router
**النجوم:** ~3,100 | **الرخصة:** MIT | **الإصدار:** v0.4.28

### ما هو المشروع؟

بوابة توجيه ذكية (Smart AI Router) تعمل محلياً، مصممة لتوفير برمجة بالذكاء الاصطناعي مجانية وغير محدودة. توفر نقطة نهاية واحدة متوافقة مع OpenAI (`/v1/*`) تُوجّه الطلبات عبر أكثر من 60 مزوّد AI مع دعم الترجمة التلقائية بين الصيغ (OpenAI ↔ Anthropic ↔ Gemini)، والانتقال الاحتياطي، وتتبع الاستخدام.

### الميزات الأساسية

- **RTK Token Saver:** ضغط تلقائي لمخرجات الأدوات → توفير 20-40% من التوكنز
- **Caveman Mode:** برومبتات مختصرة → توفير حتى 65% من توكنز المخرجات
- **3-Tier Smart Fallback:** اشتراك ← رخيص ← مجاني = صفر توقف
- **Multi-Account:** التوزيع الدائري بين عدة حسابات لكل مزوّد
- **Format Translation:** تحويل ثنائي الاتجاه بين OpenAI, Anthropic, Gemini, Cursor, Kiro, Vertex
- **Auto Token Refresh:** تجديد تلقائي لـ OAuth tokens قبل انتهائها
- **Custom Combos:** إنشاء سلاسل نماذج مخصصة غير محدودة
- **9 أنواع خدمات:** Chat, Embeddings, TTS, STT, Image, Video, Web Search, Web Fetch, Vision

### التقنيات المستخدمة

| التقنية | الغرض |
|---------|-------|
| Next.js 16 (App Router) | إطار العمل الأساسي |
| React 19.2 + Tailwind CSS 4 | الواجهة والتصميم |
| Zustand 5 | إدارة الحالة |
| SQLite (better-sqlite3 / sql.js / node:sqlite) | قاعدة بيانات ثلاثية المستويات |
| Express 5 | خادم HTTP |
| JWT (jose) + bcryptjs | مصادقة وتشفير |
| Docker (Bun Alpine) | النشر |
| Cloudflare Workers | شبكة حواف عالمية |

### البنية المعمارية

```
أدوات CLI (أي أداة برمجة)
    │
    ↓ http://localhost:20128/v1
┌─────────────────────────────────────┐
│      9Router (الراوتر الذكي)         │
│  • RTK Token Saver                  │
│  • Format Translation               │
│  • Quota Tracking                   │
│  • Auto Token Refresh               │
└──────────────┬──────────────────────┘
               │
    ├─→ [المستوى 1: اشتراك] النماذج المدفوعة
    │   ↓ نفاد الحصة
    ├─→ [المستوى 2: رخيص] نماذج منخفضة التكلفة
    │   ← تجاوز الميزانية
    └─→ [المستوى 3: مجاني] نماذج مجانية
```

### ما يناسب OmniFile_Processor 🟢 مستوى حرج

المشروع يحتوي على وحدة **AI Gateway** (80 ملف في `modules/ai/gateway/`). نمط 9router يوفر مخططاً جاهزاً ومجرباً لتعزيز هذه الوحدة:

1. **التوجيه متعدد المزودين:** دعم أكثر من 60 مزوّد مع تبديل تلقائي — يمكن تطبيقه مباشرة على بوابة OmniFile
2. **3 مستويات احتياطية:** نمط (مدفوع ← رخيص ← مجاني) يضمن عدم توقف خدمة التصحيح الإملائي
3. **حفظ التوكنز:** ضغط مخرجات OCR والسياق قبل إرسالها لنماذج AI — يوفر 20-40% من التكلفة
4. **ترجمة الصيغ:** التحويل التلقائي بين صيغ API المختلفة (OpenAI, Anthropic, Gemini)
5. **تجديد التوكنز:** ضروري للنشر في الإنتاج حيث تنتهي صلاحية بيانات API
6. **سلاسل النماذج المخصصة:** تمكين سلسلة (OCR → تصحيح → تحقق) عبر نماذج متعددة

## 2. agentmemory - ذاكرة دائمة لوكلاء الذكاء الاصطناعي

**المستودع:** https://github.com/rohitg00/agentmemory
**النجوم:** ~3,708 | **الرخصة:** Apache 2.0 | **الإصدار:** v0.9.5

### ما هو المشروع؟

نظام ذاكرة دائمة لوكلاء البرمجة بالذكاء الاصطناعي. يحل مشكلة نسيان الوكلاء لكل شيء بين الجلسات — يلتقط تلقائياً ما يفعله الوكيل، يضغطه في ذاكرة قابلة للبحث، ويعيد حقن السياق الصحيح عند بدء الجلسة التالية. يعمل عبر 16+ وكيل من خادم واحد بدون قواعد بيانات خارجية (SQLite + فهارس في الذاكرة فقط).

### الميزات الأساسية

- **4 مستويات لتكثيف الذاكرة:** عمل (ملاحظات خام) ← تجربة (ملخصات) ← دلالي (حقائق/أنماط) ← إجرائي (إجراءات/قرارات)
- **12 خطاف دورة حياة:** التقاط تلقائي لكل نشاط (بداية جلسة، إرسال برومبت، استخدام أداة، إنهاء جلسة...)
- **استرجاع ثلاثي المسارات:** BM25 (كلمات مفتاحية) + Vector (تشابه دلالي) + Graph (رسم معرفي)، مدمج بخوارزمية RRF
- **60+ دالة:** ملاحظة، بحث، ضغط، تكثيف، تذكر، عمليات رسم معرفي، استخراج دروس، فلترة خصوصية، إزالة تكرار...
- **7 مزودات تضمين:** محلي (all-MiniLM-L6-v2)، Gemini، OpenAI، Voyage AI، Cohere، OpenRouter، CLIP
- **عرض فوري:** بث مباشر للملاحظات، مستكشف الجلسات، متصفح الذاكرة، تصور الرسم المعرفي
- **تنسيق متعدد الوكلاء:** عقود إيجار، إشارات، إجراءات، مهام، مزامنة P2P
- **فلترة الخصوصية:** إزالة تلقائية لمفاتيح API والأسرار قبل التخزين
- **منحنى نسيان إبنغهاوس:** الذاكرات المتكررة تتعزز، الراكدة تُحذف تلقائياً، التناقضات تُكتشف

### التقنيات المستخدمة

| التقنية | الغرض |
|---------|-------|
| TypeScript (ESM) | اللغة الأساسية، Node.js ≥ 20 |
| iii-engine (Rust) | محرك التشغيل — ثلاثة مبادئ: Worker, Function, Trigger |
| SQLite | استمرارية الحالة |
| 827+ اختبار (vitest) | ضمان الجودة |
| 118 ملف مصدري | ~21,800 سطر |
| MCP Server | 51 أداة، 6 موارد، 3 برومبتات |
| REST API | 107 نقطة نهاية على منفذ 3111 |

### ما يناسب OmniFile_Processor 🟢 مستوى حرج

1. **نظام ذاكرة التصحيحات:** نموذج 4 مستويات يمكن تطبيقه على نظام التصحيح الإملائي:
   - **عمل:** تصحيحات الجلسة الحالية (OCR خام + تصحيحات مطبقة)
   - **تجربة:** ملخصات الجلسات (أنواع الأخطاء المصححة، أنماط مكتشفة)
   - **دلالي:** قواعد لغة عربية مستخرجة (أخطاء شائعة، أنماط التشكيل، قواعد صرفية)
   - **إجرائي:** إجراءات التصحيح (خطوات لع_types محددة من الأخطاء)

2. **فلترة الخصوصية:** ضرورية لأن OmniFile يعالج مستندات يدوية قد تحتوي على معلومات حساسة

3. **التنسيق متعدد الوكلاء:** يمكن تحسين تعامل بوابة AI مع طلبات المعالجة المتزامنة

4. **الاسترجاع الهجين (BM25 + Vector + Graph):** يمكن تحسين قدرة المدقق الإملائي على اقتراح الكلمات العربية الصحيحة بجمع المطابقة الدقيقة والتشابه الدلالي

## 3. agent-skills - مهارات هندسية إنتاجية

**المستودع:** https://github.com/addyosmani/agent-skills
**النجوم:** ~35,300 | **الرخصة:** MIT

### ما هو المشروع؟

مجموعة مهارات هندسية إنتاجية لوكلاء البرمجة بالذكاء الاصطناعي، أنشأها Addy Osmani (مدير Google Cloud AI، منشئ Chrome DevTools). تُرمّز سير العمل وبوابات الجودة وأفضل الممارسات التي يستخدمها المهندسون الكبار عند بناء البرمجيات، معبأة بطريقة تجعل وكلاء AI يتبعونها باستمرار.

### 22 مهارة عبر 6 مراحل تطوير

| المرحلة | المهارات |
|---------|---------|
| **تحديد** (DEFINE) | تحسين الأفكار، التطوير المبني على المواصفات |
| **تخطيط** (PLAN) | تفكيك المهام إلى مهام صغيرة قابلة للتحقق |
| **بناء** (BUILD) | تنفيذ تدريجي، تطوير مدفوع بالاختبارات، هندسة السياق، تطوير مصدري، تطوير مبني على الشك، هندسة الواجهات |
| **تحقق** (VERIFY) | اختبار المتصفح، التصحيح واستعادة الأخطاء |
| **مراجعة** (REVIEW) | مراجعة الكود، تبسيط الكود، الأمان والتصلب، تحسين الأداء |
| **نشر** (SHIP) | Git، CI/CD، إهمال/ترحيل، التوثيق، الإطلاق |

### الميزة الأبرز: جداول مضادة للتسويغ

كل مهارة تتضمن جدولاً بالأعذار الشائعة التي يستخدمها وكلاء AI (والمهندسون المتعبون) لتخطي خطوات الجودة، مع ردود مكتوبة مسبقاً.

### ما يناسب OmniFile_Processor 🟡 مستوى عالي

1. **معايير هندسية للمشروع:** تبني ممارسات التطوير المبني على المواصفات واختبار التطوير كمعايير للمشروع
2. **هرم الاختبارات (80/15/5):** تحسين تغطية الاختبارات (حالياً 336 اختبار)
3. **مهارات الأمان:** أنماط OWASP Top 10 قابلة للتطبيق على معالجة المستندات
4. **النمط المضاد للتسويغ:** يمكن تبنيه في ملفات تعليمات المشروع لمنع تخطي خطوات الجودة

## 4. hello-agents - بناء وكلاء AI من الصفر

**المستودع:** https://github.com/datawhalechina/hello-agents
**النجوم:** ~45,968 | **الرخصة:** CC BY-NC-SA 4.0

### ما هو المشروع؟

دورة تعليمية منهجية ومجانية ومفتوحة المصدر لتعلم تطوير وكلاء AI، من إنتاج مجتمع Datawhale. تُعلّم بناء وكلاء AI الأصليين (وليس برمجيات بسيطة تستخدم LLMs كخلفية) عبر مسار تعليمي تدريجي: من الأساسيات إلى بناء إطار عمل كامل من الصفر.

### المنهج (16 فصلاً عبر 5 أجزاء)

| الجزء | الفصول | المحتوى |
|-------|--------|---------|
| **أساسيات** | 1-3 | تعريف الوكلاء، التاريخ، أساسيات LLM (Transformer, prompts) |
| **بناء الوكلاء** | 4-7 | ReAct/Plan-and-Solve/Reflection عملي، منصات low-code (Coze, Dify, n8n)، أطر عمل (LangGraph, AutoGen)، بناء إطار عمل من الصفر |
| **متقدم** | 8-12 | الذاكرة و RAG، هندسة السياق، بروتوكولات التواصل (MCP, A2A, ANP)، تعلم RL للوكلاء (SFT→GRPO)، تقييم الأداء |
| **حالات دراسية** | 13-15 | مساعد سفر ذكي (MCP + multi-agent)، وكيل بحث عميق، بلدة سبرانية |
| **مشروع ختامي** | 16 | بناء تطبيق multi-agent كامل |

### ما يناسب OmniFile_Processor 🟡 مستوى عالي

1. **بروتوكول MCP (الفصل 10):** توحيد التواصل بين محركات OCR والمدقق الإملائي ونماذج AI
2. **هندسة السياق (الفصل 9):** إدارة سياق المحادثة لسير عمل التصحيح متعدد الجولات
3. **تعلم RL للوكلاء (الفصل 11):** ضبط دقيق لنموذج التصحيح العربي (SFT→GRPO)
4. **الذاكرة و RAG (الفصل 8):** تحسين نظام التصحيح بأساليب الاسترجاع المعزز
5. **الإطار المبني من الصفر (الفصل 7):** مرجع معماري لوحدة AI Gateway

## 5. financial-services - وكلاء الخدمات المالية

**المستودع:** https://github.com/anthropics/financial-services
**النجوم:** ~17,800 | **الرخصة:** Apache 2.0

### ما هو المشروع؟

مجموعة رسمية مفتوحة المصدر من Anthropic تحتوي على وكلاء AI ومهارات وموصلات بيانات مصممة خصيصاً لسير عمل الخدمات المالية: 10 قوالب وكلاء مسمى، 7 حزم مهارات عمودية، 50+ مهارة/أمر، و11 تكامل MCP مع مزودي بيانات مالية.

### 10 قوالب وكلاء

1. **Pitch Agent:** إنشاء عروض تقديمية مع تحليلات المقارنة
2. **Meeting Prep Agent:** تحضير ملفات تعريفية قبل الاجتماعات
3. **Market Researcher:** بحث قطاعي وشامل
4. **Earnings Reviewer:** مراجعة الأرباح وتحديث النماذج
5. **Model Builder:** بناء نماذج مالية (DCF, LBO, 3-statement)
6. **Valuation Reviewer:** مراجعة التقييمات
7. **GL Reconciler:** تسوية الدفاتر العامة
8. **Month-End Closer:** إقفال الشهر
9. **Statement Auditor:** تدقيق البيانات المالية
10. **KYC Screener:** فحص العميل

### الأنماط المعمارية المهمة

- **نشر مزدوج:** نفس الكود يعمل تفاعلياً أو كـ API
- **تنسيق متعدد الوكلاء (عمق 1):** منسق ← باحث + مصمم + كاتب (المستوى الأوروبي فقط يكتب)
- **الإنسان في الحلقة:** الوكلاء يُعدّون المسودات فقط — لا يتخذون قرارات مستقلة
- **بنية قائمة على الملفات:** Markdown/YAML/JSON بدون خطوة بناء

### ما يناسب OmniFile_Processor 🟡 مستوى متوسط

1. **نمط التنسيق (عمق 1):** يمكن تحسين تعامل البوابة مع طلبات OCR المعقدة
2. **الإنسان في الحلقة:** ضروري لنظام تصحيح إملائي يحتاج مراجعة المستخدم
3. **تنظيم حزم المهارات:** يمكن تنظيم قواعد التصحيح حسب المجال اللغوي
4. **مسار التدقيق:** تتبع كل تصحيح على مخرجات OCR لضمان الجودة

## 6. Dive-into-LLMs - دورة نماذج اللغة الكبيرة

**المستودع:** https://github.com/aiwaves-cn/Dive-into-LLMs
**النجوم:** ~10 | **الرخصة:** Apache 2.0

### ما هو المشروع؟

المستودع الرسمي لدورة تعليمية مفتوحة عبر الإنترنت عن نماذج اللغة الكبيرة، من إنتاج AIWaves Inc. الدورة تُدرّس بالصينية وتغطي أساسيات OpenAI GPT API.

### حالة المشروع

المشروع في مرحلة مبكرة جداً — دفتر واحد فقط منشور ( Lecture 1 ) يغطي أساسيات OpenAI API، ولم يحدث منذ ~14 شهراً. المحتوى الموجود جيد لكن محدود.

### ما يناسب OmniFile_Processor 🟡 مستوى متوسط

قيمة تعليمية محدودة — فهم أساسيات API مفيد لوحدة AI Gateway، لكن hello-agents (46K نجمة) يوفر محتوى أعمق بكثير.

## 7. UI-TARS-desktop - أتمتة واجهة المستخدم

**المستودع:** https://github.com/bytedance/UI-TARS-desktop
**النجوم:** ~23,000 | **الرخصة:** Apache 2.0

### ما هو المشروع؟

حزمة وكلاء AI متعدد الوسائط مفتوحة المصدر من ByteDance تمكّن AI من إدراك وفهم والتحكم في واجهات المستخدم الرسومية (سطح المكتب، المتصفحات، الأطرافية) باستخدام تعليمات اللغة الطبيعية. يشمل نموذج رؤية-لغة (VLM) بـ 2B/7B/72B معلمة.

### ما يناسب OmniFile_Processor 🔵 مستوى منخفض

التطبيق المباشر محدود لأن المشروع متخصص في أتمتة واجهات المستخدم الرسومية. لكن بعض الأنماط غير المباشرة مفيدة:
- النهج الهجين (رؤية + قواعد) يمكن إلهام نهج OCR هجين
- بنية MCP القابلة للتوسع مرجع جيد للبوابة
- التنفيذ المعزول (sandbox) مفيد لتنفيذ سكريبتات المستخدم بأمان
