### 2️⃣ **AgentMemory / OpenMemory** - الذاكرة الدائمة المحلية
| الميزة | التفاصيل | 💡 ما يناسب مشروعك |
|--------|----------|-------------------|
| 🧠 **ذاكرة معرفية متعددة القطاعات** | ذاكرة حلقيّة، دلالية، إجرائية، عاطفية، تأملية | ✅ **مفيد جداً**: لحفظ أنماط الأخطاء الشائعة في نصوصك الطبية/العربية وتحسين التصحيح التلقائي |
| 💾 **محلي بالكامل (SQLite)** | لا يحتاج API Keys، يعمل أوفلاين | ✅ **ممتاز**: يتوافق مع تفضيلك للمعالجة المحلية |
| 🔍 **استرجاع قابل للتفسير** | يوضح لماذا تم استرجاع معلومة معينة | ✅ **مفيد**: لتتبع قرارات التصحيح الإملائي أو الترجمة |

**🔧 طريقة الدمج المقترحة**:
```python
# في وحدات nlp/spell_corrector.py و ai/pattern_matcher.py
from openmemory.client import Memory

# إنشاء ذاكرة خاصة بأنماط التصحيح الطبية
medical_memory = Memory(db_path="data/medical_corrections.db")

# عند تصحيح كلمة:
async def learn_from_correction(original, corrected, context):
    await medical_memory.add(
        f"Pattern: '{original}' → '{corrected}' in medical context: {context}",
        user_id="ocr_system",
        sector="procedural"  # نمط إجرائي للتعلم
    )

# عند معالجة نص جديد:
async def suggest_correction(word, context):
    results = await medical_memory.search(
        f"medical term similar to: {word}",
        user_id="ocr_system",
        limit=3
    )
    return results  # أنماط مشابهة تم تعلمها مسبقاً
```

**🔧 طريقة الدمج المقترحة**:
```markdown
# modules/nlp/skills/arabic-ocr-correction/SKILL.md
---
name: arabic-ocr-correction
description: Guides correction of Arabic OCR output with medical terminology protection
when_to_use: After OCR extraction when text contains Arabic medical terms
---

## Process
1. **Identify domain**: Detect if text is medical/legal/literary using language_detector.py
2. **Protect terms**: Skip correction for terms in medical_term_dictionary.json
3. **Apply rules**: Use arabic_rtl.py normalization mappings (40+ rules)
4. **Verify**: Check CER/WER against ground truth if available
5. **Learn**: Save successful corrections to correction_dict.json

## Anti-Rationalizations
| Excuse | Rebuttal |
|--------|----------|
| "I'll fix it later" | Errors compound in batch processing; fix now |
| "The model knows best" | TrOCR struggles with Arabic ligatures; rules are essential |

## Verification
- [ ] CER < 5% for Arabic medical texts
- [ ] No protected terms were altered
- [ ] RTL rendering preserved in output
```

## ⚠️ ملاحظات هامة قبل الدمج

1. **التوافق مع البنية الحالية**: مشروعك يستخدم بنية مزدوجة (`src/` عملي، `modules/` نظري). ابدأ بالدمج في `modules/` ثم هاجر تدريجياً.

2. **الأوفلاين أولاً**: بما أنك تفضل المعالجة المحلية، تأكد أن أي مكتبة تضيفها تعمل بدون اتصال (AgentMemory ✅، 9Router يحتاج إعداداً محلياً ✅).

3. **اللغة العربية**: تأكد أن أي أداة ذاكرة أو معالجة تدعم:
   - النص من اليمين لليسار (RTL)
   - التشكيل والربط العربي
   - المصطلحات الطبية ثنائية اللغة

4. **التراخيص**: جميع المستودعات المذكورة تستخدم تراخيص مفتوحة (MIT, Apache 2.0) مما يسمح بالدمج في مشروعك المرخص بـ MIT.
