```
الهدف: دعم تبديل تلقائي بين مزودي AI المختلفين
الموقع: modules/ai/gateway/
```

- [ ] دعم نقطة نهاية واحدة متوافقة مع OpenAI (`/v1/*`)

  - **المستوى 2 (رخيص):** GLM، MiniMax، DeepSeek
  - **المستوى 3 (مجاني):** نماذج محلية (Ollama)، Vertex ($300 credits)
- [ ] توزيع دائري (round-robin) بين عدة مفاتيح API
- [ ] كشف تلقائي لصيغة المزوّد وترجمتها

- [ ] ضغط سياق التصحيح (الكلمات المحيطة)

#### 1.4 تتبع الاستخدام والتحليلات

## المرحلة 2: نظام ذاكرة التصحيحات

**المصدر:** agentmemory ([agentmemory/](agentmemory/))
**المدة:** 3-4 أسابيع
**الأثر:** تحسين مستمر ودائم في دقة التصحيح الإملائي

### المهام

#### 2.1 نماذج الذاكرة الأربعة
```
الهدف: بناء ذاكرة متعددة المستويات للتصحيحات
الموقع: modules/spellcheck/memory/
```

- [ ] **ذاكرة العمل (Working):** تخزين مؤقت لتصحيحات الجلسة الحالية
  - ملاحظات خام: (كلمة OCR الأصلية ← التصحيح المطبق)
  - دورة حياة: تنتهي بنهاية الجلسة
- [ ] **ذاكرة تجريبية (Episodic):** ملخصات الجلسات السابقة
  - أنواع الأخطاء المصححة في كل جلسة
  - الأنماط المكتشفة
  - نسبة الدقة
- [ ] **ذاكرة دلالية (Semantic):** قواعد لغة عربية مستخرجة
  - أخطاء إملائية شائعة وتصحيحاتها
  - أنماط التشكيل والحركات
  - قواعد صرفية (جذر ← مشتقات)
- [ ] **ذاكرة إجرائية (Procedural):** إجراءات التصحيح
  - خطوات معالجة أنواع محددة من الأخطاء
  - سلاسل القرارات للتصحيح المعقد
  - استراتيجيات الأولوية

#### 2.2 فلترة الخصوصية
```
الهدف: حماية محتوى المستندات الحساسة
الموقع: modules/spellcheck/memory/privacy.py
```

- [ ] إزالة تلقائية للأسماء الشخصية قبل التخزين
- [ ] فلترة الأرقام (هواتف، بطاقات، حسابات)
- [ ] إزالة علامات `<private>` من النص
- [ ] تشفير المحتوى المخزن

#### 2.3 الاسترجاع الهجين
```
الهدف: البحث الذكي في ذاكرة التصحيحات
الموقع: modules/spellcheck/memory/retrieval.py
```

- [ ] بحث BM25 (كلمات مفتاحية عربية مع stemmer)
- [ ] بحث Vector (تشابه دلالي عبر embeddings محلية)
- [ ] رسم معرفي (علاقات بين الكلمات العربية)
- [ ] دمج النتائج بخوارزمية RRF (Reciprocal Rank Fusion)

#### 2.4 منحنى النسيان
```
الهدف: تحسين الذاكرة تلقائياً بمرور الوقت
الموقع: modules/spellcheck/memory/decay.py
```

- [ ] تعزيز القواعد الأكثر استخداماً
- [ ] إزالة تلقائية للقواعد الراكدة (TTL)
- [ ] كشف التناقضات وحلها (عند تعارض قاعدتين)
- [ ] ضبط أوزان الأهمية

## المرحلة 3: بروتوكولات التواصل والتكامل

**المصدر:** hello-agents ([hello-agents/](hello-agents/))
**المدة:** 2-3 أسابيع
**الأثر:** توحيد وتحسين التواصل بين مكونات النظام

### المهام

#### 3.1 تبني بروتوكول MCP
```
الهدف: توحيد التواصل بين جميع المكونات عبر MCP
الموقع: modules/ai/gateway/mcp/
```

- [ ] تحويل كل محرك OCR إلى MCP Server منفصل:
  - `TrOCR-MCP-Server` (نموذج الرؤية)
  - `EasyOCR-MCP-Server` (محرك متعدد اللغات)
  - `Tesseract-MCP-Server` (محرك مفتوح المصدر)
- [ ] تحويل المدقق الإملائي إلى MCP Server:
  - `SpellCheck-MCP-Server` (التحقق والإصلاح)
  - `Dictionary-MCP-Server` (القاموس ثنائي اللغة)
- [ ] تعريف أدوات (tools) واضحة لكل خادم:
  - `ocr_recognize_text` (التعرف على النص)
  - `ocr_detect_regions` (كشف مناطق النص)
  - `spell_check_word` (التحقق من كلمة)
  - `spell_suggest_corrections` (اقتراح تصحيحات)
  - `spell_apply_batch` (تطبيق تصحيحات جماعية)
- [ ] تعريف موارد (resources) للمشاركة:
  - قاموس التصحيحات
  - نماذج اللغة العربية
  - إعدادات المعالجة

#### 3.2 هندسة السياق
```
الهدف: إدارة ذكية لسياق المحادثات متعددة الجولات
الموقع: modules/ai/gateway/context/
```

- [ ] بناء مدير سياق يحدد المعلومات المطلوبة لكل خطوة
- [ ] حقن الذاكرة ذات الصلة تلقائياً عند بدء الجلسة
- [ ] ضغط السياق عندما يتجاوز حد التوكنز
- [ ] أولوية المعلومات (التصحيحات الأخيرة > الأنماط الدلالية > القواعد العامة)

#### 3.3 ضبط دقيق للنموذج (Agentic RL)
```
الهدف: تحسين دقة التصحيح العربي عبر الضبط الدقيق
الموقع: modules/ai/gateway/training/
```

- [ ] تجهيز بيانات التدريب من ذاكرة التصحيحات المتراكمة
- [ ] تنفيذ خط أنابيب SFT (Supervised Fine-Tuning)
- [ ] تنفيذ خط أنابيب GRPO (Group Relative Policy Optimization)
- [ ] تقييم الأداء قبل وبعد الضبط (WER/CER metrics)
