# الاقتراحات المُرتبة والمُعتمدة — OmniFile AI Processor

هذا الملف يجمع الاقتراحات المناسبة للمشروع بعد مراجعة `SUGGESTIONS.md` والمرفقات، ثم يرتبها بحسب الأولوية وقابلية الدمج.

## ما تم اعتماده ودمجه بالفعل

### أولوية عالية — تم التنفيذ
1. **تحسين الاستقرار والاختبارات الأساسية**
   - إصلاحات في التقييم، كشف الاتجاه، الاستيراد الاختياري، ومعالجة الصور.
2. **ملف متطلبات خاص بـ Google Colab**
   - إضافة `requirements-colab.txt` لتشغيل الدفاتر بسهولة.
3. **دفتر Colab للتجريب والتصحيح**
   - إضافة دفتر مخصص لتشغيل المشروع ومراجعته من Colab.
4. **أداة بناء بيانات تدريب من التصحيحات**
   - إضافة `tools/build_training_data.py` لتحويل نتائج المراجعة اليدوية إلى JSONL.
5. **مسار تجريبي لـ Hugging Face Spaces**
   - تجهيز نسخة خفيفة مخصصة للنشر كمساحة Gradio مستقلة.

## اقتراحات مناسبة للمشروع — المرحلة التالية

### P1 — تحسينات قريبة التنفيذ
1. **Benchmarks واقعية**
   - قياس السرعة والدقة لكل محرك OCR على عينات عربية/إنجليزية/ألمانية.
2. **Progress bars أوضح في الواجهات**
   - إبراز مراحل المعالجة: preprocessing → OCR → NLP → export.
3. **تحسين وثائق الاعتماديات**
   - توضيح الفروق بين `requirements-core.txt` و `requirements-ocr.txt` و `requirements-nlp.txt` و `requirements-full.txt`.
4. **تحسين دعم Colab والـ mobile review**
   - ربط مخرجات المراجعة المحمولة مباشرة بأداة إعداد البيانات التدريبية.

### P2 — تحسينات متوسطة الأجل
1. **Layout-preserving mode موسّع**
   - تطوير التصدير مع الحفاظ الأفضل على الجداول والتسميات والصور.
2. **Collaborative review**
   - دعم جلسات مراجعة متعددة أو مزامنة أخف لفرق التحرير.
3. **لوحة أداء مصغرة**
   - إحصاءات على نتائج OCR، نسب التصحيح، ومخرجات التقييم.
4. **مراقبة ونشر**
   - Prometheus/Grafana أو بديل أخف لمراقبة خدمات الخلفية.

### P3 — تحسينات مستقبلية
1. **Table Transformer / LayoutLM**
   - تحسين فهم بنية الصفحة والجداول للمستندات المعقدة.
2. **RAG / QA على المستندات**
   - أسئلة وأجوبة فوق النصوص المعالجة.
3. **توسيع اللغات**
   - إضافة الفرنسية أو الإسبانية بعد تثبيت خط الأساس الحالي.

## اقتراحات تمت مراجعتها ولم تُعتمد حالياً
- **زيادة التعقيد السريع في الواجهة**: تم تأجيل أي توسع كبير يضيف عبئاً قبل تثبيت الأساس.
- **نشر سحابي معقد فوراً**: الأولوية الآن للاستقرار وقابلية التجربة، ثم التوسع.
- **إعادة بناء كاملة للواجهة**: غير ضرورية حالياً مع وجود Streamlit وGradio وReact.

## معايير الاعتماد
يُعتمد الاقتراح إذا كان:
1. يحل مشكلة فعلية أو ثغرة استقرار.
2. يضيف قيمة مباشرة للتجريب أو التدريب أو النشر.
3. يمكن دمجه بدون كسر البنية الحالية.
4. لا يضاعف الاعتماديات الثقيلة بلا حاجة.
